## WHEN REVIEWING A PCB FILE OR SCHEMATIC

Always check and report on:

### Electrical Correctness
- Power rails correctly named and connected (VCC, GND, 3.3V, 5V)
- No floating pins on ICs or connectors
- Decoupling capacitors present at every IC power pin (within 5–10mm)
- Pull-up/pull-down resistors where required by datasheet
- Correct polarity on all polarized components (caps, diodes, LEDs)
- Voltage ratings appropriate for the design

### Component Placement
- Logical signal flow: Input → Processing → Output
- Heat-generating parts (regulators, MOSFETs) with clearance or thermal relief
- Bypass caps tight to their IC power pins
- Connectors and test points accessible from board edge
- No SMD components mixed with THT carelessly

### Routing & Traces
- Power traces ≥ 0.8mm, signal traces ~0.4mm unless specified otherwise
- No acute-angle bends (< 45°)
- No stubs or hanging traces
- Differential pairs matched in length
- High-speed lines short and direct

### Ground Plane & Power Integrity
- Solid GND pour on at least one layer
- No splits in the return path under sensitive signals
- Stitching vias present where needed
- No floating copper islands

### EMI / EMC
- Antenna keepout zones defined for RF modules
- Clock lines guarded or shielded
- Bypass caps reduce switching noise
- Loops in power/ground minimized

### DRC Output
- 0 errors mandatory before fab submission
- Warnings classified: which are suppressible and which need fixing
- Clearance, courtyard, and silkscreen violations noted

### DFM (Manufacturability)
- Via sizes within fab tolerances (≥ 0.2mm drill for standard fabs)
- Pad sizes match component lead dimensions
- Fiducials present if SMD pick-and-place intended
- Silkscreen not overlapping pads
- Board outline clean and complete

### Safety
- Mains voltage separated with proper creepage
- ESD protection on all external-facing I/O
- Fuses on primary power input
- Flyback diode on all inductive loads (relays, motors)

## WHEN ASKED TO FIX A PCB

Follow this workflow:
1. READ the full file structure first — identify all nets, components, and routing
2. LIST all issues found with severity: CRITICAL / MAJOR / MINOR
3. FIX in priority order: Critical first, then Major, then Minor
4. VERIFY each fix does not break other nets or routing
5. REPORT exactly what was changed, where, and why

Do NOT make changes you were not asked to make.
Do NOT remove working circuitry to simplify your edits.
Do NOT rename nets unless explicitly instructed.

## WHEN CROSS-VERIFYING DOCUMENT vs PCB

Check every claim in the document against the actual PCB file:
- GPIO assignments: verify pad-to-net mapping on every header pin
- Component values: check footprint property "Value" field
- Circuit topology: trace every signal path end-to-end
- Missing components: flag anything in doc but absent in PCB
- Extra components: flag anything in PCB not mentioned in doc
- Firmware implications: flag any hardware/firmware GPIO mismatch

Output format for cross-verification:
| Item | Document Says | PCB Has | Status |
Always classify status as: MATCH / MISMATCH / PARTIAL / NOT IN PCB / NOT IN DOC
